By the end of this guide, you'll have the Aden Agent Framework installed and run your first goal-driven agent.

## Prerequisites

<CardGroup cols={2}>
  <Card title="Python" icon="python">
    Python 3.11 or higher (3.11, 3.12, or 3.13)
  </Card>
  <Card title="LLM Provider" icon="key">
    API key for Anthropic (recommended) or OpenAI
  </Card>
</CardGroup>

## Step 1: Clone the Repository

```bash
git clone https://github.com/adenhq/hive
cd hive
```

## Step 2: Run the Setup Script

The automated setup script handles all dependencies:

```bash
./scripts/setup-python.sh
```

This will:
- Check your Python version
- Install the core framework package (`framework`)
- Install the tools package (`aden_tools`)
- Fix package compatibility issues
- Verify all installations

<Accordion title="Manual Setup (Alternative)">
If you prefer to set up manually or the script fails:

**Install Core Framework:**
```bash
cd core && pip install -e .
```

**Install Tools Package:**
```bash
cd tools && pip install -e .
```

**Upgrade OpenAI Package:**
```bash
pip install --upgrade "openai>=1.0.0"
```

**Verify Installation:**
```bash
python -c "import framework; print('✓ framework OK')"
python -c "import aden_tools; print('✓ aden_tools OK')"
python -c "import litellm; print('✓ litellm OK')"
```
</Accordion>

## Step 3: Set Up Environment

Create a `.env` file or export your API key:

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
```

<Info>
For running agents with real LLMs, you'll need an API key from Anthropic or OpenAI. You can also use `--mock` mode for testing without API calls.
</Info>

## Step 4: Run Your First Agent

All agent commands must be run from the project root with `PYTHONPATH` set:

```bash
# Validate agent structure
PYTHONPATH=core:exports python -m support_ticket_agent validate

# Show agent information
PYTHONPATH=core:exports python -m support_ticket_agent info
```

Run the agent with sample input:

```bash
PYTHONPATH=core:exports python -m support_ticket_agent run --input '{
  "ticket_content": "My login is broken. Error 401.",
  "customer_id": "CUST-123",
  "ticket_id": "TKT-456"
}'
```

<Tip>
Use `--mock` flag to test agents without making LLM API calls:
```bash
PYTHONPATH=core:exports python -m support_ticket_agent run --mock --input '{...}'
```
</Tip>

## Step 5: Explore Other Agents

The framework includes several example agents:

```bash
# Market Research Agent
PYTHONPATH=core:exports python -m market_research_agent info

# Outbound Sales Agent
PYTHONPATH=core:exports python -m outbound_sales_agent validate

# Personal Assistant Agent
PYTHONPATH=core:exports python -m personal_assistant_agent info
```

## You're Ready

You now have a working agent development environment. Your agents can process inputs, make LLM calls, and execute complex workflows.
